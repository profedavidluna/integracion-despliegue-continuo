# Módulo II: Fundamentos de CI/CD y DevOps

## Descripción General

Este módulo construye sobre los contenedores del Módulo I para enseñar la capa cultural y de automatización que los rodea: **DevOps y los pipelines de CI/CD**. Aprenderás a pensar como una organización de alto rendimiento, a construir pipelines que automaticen la entrega de software y a configurar entornos reproducibles que eliminen la fricción entre equipos. Al finalizar, comprenderás por qué las mejores empresas del mundo despliegan cientos de veces al día y cómo replicar esas prácticas en tu organización.

---

## Objetivos de Aprendizaje

Al completar este módulo, el estudiante será capaz de:

- Explicar el valor de negocio de DevOps con datos empíricos del reporte DORA.
- Describir el conflicto tradicional Dev vs. Ops y cómo DevOps lo resuelve.
- Aplicar los Tres Caminos de DevOps (Flujo, Feedback, Aprendizaje) como marco conceptual.
- Diferenciar con precisión Integración Continua, Entrega Continua y Despliegue Continuo.
- Configurar un pipeline básico en Jenkins y en GitHub Actions.
- Identificar las buenas prácticas de CI más importantes (Trunk-Based Development, Fail Fast, Pipeline as Code).
- Explicar qué es Infraestructura como Código y por qué es fundamental para la paridad de entornos.
- Seleccionar recursos de aprendizaje adicionales para profundizar en cada tema.
